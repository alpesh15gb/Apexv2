# Competitor Research: Enterprise HRMS Platforms

**Date**: 2026-06-26  
**Researcher**: Principal Product Designer

---

## Executive Summary

Analysis of 12 leading HRMS platforms reveals common UX patterns that make enterprise software feel premium. The key insight: **successful HRMS products are designed around workflows, not database schemas**.

---

## 1. Keka HR

### Navigation
- **Sidebar**: Collapsible, grouped by function (People, Time, Payroll, Performance)
- **Topbar**: Search, notifications, company switcher, profile
- **Active states**: Bold text + colored background
- **Icons**: Consistent, minimal, purposeful

### Dashboard
- **Layout**: 2x3 KPI grid + charts + pending items
- **KPIs**: Compact cards with trend indicators
- **Charts**: Attendance trend, department distribution
- **Pending work**: Leaves to approve, missing punches

### Employee Module
- **List**: Table with photo, name, department, status
- **Profile**: Tabbed (Overview, Attendance, Leave, Documents)
- **Search**: Instant search with filters
- **Bulk actions**: Select multiple → bulk operations

### Why it works
Keka optimizes for the **daily HR workflow**. The dashboard shows exactly what needs attention today. Employee list is dense and fast for power users.

## Key Patterns Identified

### Navigation Patterns
1. **Sidebar**: Always present, collapsible, grouped by function
2. **Topbar**: Search, notifications, profile, company info
3. **Command palette**: Ctrl+K for power users
4. **Breadcrumbs**: For deep navigation
5. **Mobile**: Bottom navigation for mobile

### Dashboard Patterns
1. **KPIs**: 4-7 compact cards with today's metrics
2. **Charts**: Attendance trend, department distribution
3. **Pending work**: Leaves, missing punches, approvals
4. **Quick actions**: Common tasks accessible
5. **Activity feed**: Recent activity

### Employee Patterns
1. **List**: Dense table with search, filters, bulk actions
2. **Profile**: Tabbed layout (Overview, Attendance, Leave, Documents)
3. **Directory**: Searchable, filterable
4. **Organization**: Visual org chart

### Attendance Patterns
1. **Calendar**: Monthly view with status dots
2. **Timeline**: Daily punch timeline
3. **Exceptions**: Missing punches, late arrivals
4. **Analytics**: Trends, comparisons

### Design Patterns
1. **Typography**: Strong hierarchy (24px titles, 13px body)
2. **Spacing**: Tight, information-dense
3. **Colors**: Minimal, purposeful
4. **Cards**: Flat, border-only
5. **Tables**: Dense, sortable, filterable
